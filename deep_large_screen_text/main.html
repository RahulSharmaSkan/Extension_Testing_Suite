🌲 Deep & Large Screen Text

Generate a page with a controllable DOM nesting depth and total size to exercise deep / large screen_text capture (e.g. JSON recursion-depth limits).

Number of nested elements from root to deepest leaf (1 – 10,000).
Extra sibling leaf nodes added at each depth level (1 – 100). 1 = a pure deep spine. Node count grows ≈ depth × breadth (does not change nesting depth).
Fill-to target for total rendered HTML/DOM size (1 – 100 MB). The depth × breadth skeleton is padded up to this size; if the skeleton is already larger, this is just a floor and the actual size is reported.
Set the depth, breadth and size, then click Generate page.

How it works: "Generate page" builds a real same-origin URL (generated.html?depth=…&breadth=…&size=…). "Go to page" opens that page in a new tab, where a "spine" of nested elements is built to the chosen depth, with breadth sibling nodes at each level, then padded with content blocks until the target size is reached. Because it is a real https page (not a blob: URL), browser extensions capture it normally.

Warning: very high depth or size values (e.g. > 3,000 depth or > 25 MB) can freeze or crash the browser tab. You will be asked to confirm before generating.
